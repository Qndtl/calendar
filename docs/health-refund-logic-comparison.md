# 건강보험 공제/환급 로직 비교: 기존 vs 신규

## 용어 정리

| 코드 변수 | 의미 | 오프셋 |
|---|---|---|
| `twoMonthsAgo` | 5개월 전 | -2 |
| `oneMonthAgo` | 4개월 전 | -1 |
| `currentMonth` | 3개월 전 | 0 |
| `oneMonthAfter` | 2개월 전 | +1 |
| `twoMonthsAfter` | 1개월 전 | +2 |

### 기간 정의 (신규 환급 로직에서 사용)

- **4개월 전 기간**: 4개월전 첫출역일 ~ +30일
  - +30일 = 첫출역일 + (해당월 마지막일 - 1)
  - 예외: 첫출역일이 초일이면 → 해당월 말일까지
- **3개월 전 기간 (from-start)**: 3개월전 첫출역일 ~ +30일
  - +30일 = 첫출역일 + (해당월 마지막일 - 1)
  - window가 2개월전 월까지 걸칠 수 있음 (allDates에 sorted2 포함)

## 1. 공제 로직

### 기존 공제 플로우 (existing.js)

```
calculateHealthInsuranceDeduction
│
├─ A. 전월 없음
│   └─ 당월 초일 출역? → 당월 단독 판단
│       당월 초일 미출역? → 비공제
│
├─ B. 전월 있음
│   ├─ B-1. 전전월 있음 → 당월 단독 판단
│   ├─ B-2. 전월 초일 출역 → 당월 단독 판단
│   ├─ B-3. 전월 첫출역 1/30 or 1/31
│   │       merged = 전월 + 당월
│   │       → merged[7] 공제 (합산 기준)
│   └─ B-4. 일반 (크로스-월 윈도우)
│           merged = 전월 + 당월
│           windowEnd = 전월 첫출역 + 30일
│           filtered = merged 중 <= windowEnd
│           ├─ filtered >= 8 → merged[7] 공제
│           └─ filtered < 8  → 당월 단독 판단
│
└─ [당월 단독 판단]
    ├─ 당월 >= 8 → 당월[7] 공제, 초과분 추가 공제
    └─ 당월 < 8  → 비공제
```

**특징**: 분기 7개, 전전월·초일·1/30-31 각각 별도 처리

---

### 신규 공제 플로우 (new.js)

```
calculateHealthInsuranceDeduction
│
├─ STEP 1. 전월 없음
│   ├─ 당월 초일 출역 X → 비공제
│   └─ 당월 초일 출역 O → [당월 단독 판단]
│
└─ STEP 2. 전월 있음 → [당월 단독 판단]

[당월 단독 판단]
├─ 당월 >= 8 → 당월[7] 공제, 초과분 추가 공제
└─ 당월 < 8  → 비공제
```

**특징**: 전월 유무 + 초일 체크만으로 판단. 크로스-월 윈도우 없음

**특징**: 분기 경로 ~25개, 서브 함수 8개, 엣지케이스별 개별 분기, B·C 중복 패턴 존재

### 신규 환급 플로우 (new.js v2)

```
calculateHealthInsuranceRefund
│
│  [사용 데이터]
│  sorted4 = 4개월전 출역 (oneMonthAgo)
│  sorted3 = 3개월전 출역 (currentMonth)
│  sorted2 = 2개월전 출역 (oneMonthAfter)
│  allDates = sorted4 + sorted3 + sorted2 (전체)
│
├─ Step 1. 합산 < 8일
│   합산 기준: sorted4=0이면 sorted3+sorted2, 아니면 sorted4+sorted3
│   └─ 공제내역 있으면 환급, 없으면 비대상
│
└─ Step 2. 합산 >= 8일
    │
    │  [기간 계산]
    │  4개월전 기간: sorted4[0] ~ windowEnd4
    │    windowEnd4 = sorted4[0] + 30일
    │    (초일 출역이면 해당월 말일)
    │  period4Count = allDates 중 기간 내 날짜 수
    │
    │  3개월전 기간 (from-start): sorted3[0] ~ windowEnd3
    │    windowEnd3 = sorted3[0] + (해당월 일수 - 1)일
    │    (sorted2 날짜가 window 안에 포함될 수 있음)
    │  period3Count = allDates 중 기간 내 날짜 수
    │
    ├─ Step 3-4. 모든 기간 < 8일 (period4, period3 모두)
    │   └─ 공제내역 있으면 환급, 없으면 비대상
    │
    └─ Step 5. 하나라도 >= 8일
        │
        ├─ 5a. 4개월전 기간 >= 8일
        │   ├─ 공제 정당 조건 (하나 이상 충족)
        │   │   ① 기간 종료일 당일 또는 이후 출역 (`allDates.some(d >= period4End)`)
        │   │   ② sorted4가 1/30·31 + sorted3 말일 출역 (`sorted3LastDayWorked`)
        │   │   ③ sorted5(5개월전) 출역 있음 → 연속근로 인정 (`twoMonthsAgo.length > 0`)
        │   │   → 공제 정당
        │   │   ├─ 공제내역 있음 → 금액비교
        │   │   └─ 공제내역 없음 → 징수 (3개월전 출역 날짜 전체)
        │   └─ 위 조건 모두 미충족
        │       → 공제 부당 → 환급
        │
        ├─ 5b. 4개월전 기간 < 8일 → sorted4 유무 체크
        │   ├─ sorted4>0 (4개월전 있음) → 공제 정당 (sorted5/초일 여부 무관)
        │   │   ├─ sorted3 >= 8일 → 금액비교 / 징수
        │   │   └─ sorted3 < 8일 → 환급
        │   └─ sorted4=0 (4개월전 없음) → Step 5c/5d로
        │       ※ sorted5 있어도 sorted4 없으면 공제 정당 근거 없음
        │
        ├─ 5c. 3개월전 기간 >= 8일 ※sorted4=0일 때만 도달
        │   ├─ conditionA 충족: sorted3 초일 출역 + (말일 출역 OR sorted2 있음)
        │   │   → 공제 정당
        │   │   ├─ 공제내역 있음 → 금액비교
        │   │   └─ 공제내역 없음 → 징수 (3개월전 출역 날짜 전체)
        │   └─ conditionA 미충족 (초일 미출역 또는 연속성 없음)
        │       → 환급
        │
        └─ 5d. 3개월전 기간 < 8일
            └─ 무조건 환급
```

**특징**: 분기 경로 ~12개, 서브 함수 없음, 기간 개념으로 엣지케이스 흡수, 조회 범위 2개월전 말일까지 확장  
**반환값**: `{ refunds: [], deducts: [] }` — 공제 정당인데 공제내역 없으면 `deducts`에 징수 날짜 포함

## 3. 기존 엣지케이스가 신규 로직에서 흡수되는 방식

### 초일 출역 (기존 A-1, C)

- **기존**: 별도 분기로 처리
- **신규**: 기간 정의에서 흡수 — "첫출역이 초일이면 windowEnd = 해당월 말일" → 4개월전 기간이 해당월 전체. 별도 분기 불필요

### 1/30, 1/31 특수일 (기존 A-2a, D)

- **기존**: 별도 분기로 처리 (1월→2월 월경계 특수처리)
- **신규**: +30일 계산이 자연스럽게 2월말로 넘어감 (1/30 + 30일 = 2/28 or 29). 월경계를 기간 계산이 자연 처리

### 5개월전 출역 유무 (기존 B)

- **기존**: 별도 분기 — 5개월전 있으면 3개월전만 체크
- **신규**: sorted4 있음 여부만 체크. sorted5는 별도 분기 없이 sorted4 조건에 흡수됨
  - sorted4 있음 → 공제 정당 (sorted5 유무 무관)
  - sorted4 없음 → Step 5c로 (sorted5 있어도 공제 정당 근거 없음)

## 4. 기존 대비 결과가 달라지는 케이스

### Case 1: 환급 확인 범위 확장 (기존 버그 수정)

**시나리오**: 4개월전 기간 >= 8일, +30일 미출역, +31일 이후 출역이 2개월전에만 있음

| | 기존 | 신규 |
|---|---|---|
| 확인 범위 | +31일 ~ **3개월전 말일** | +31일 ~ **2개월전 말일** |
| 결과 | 2개월전 출역 못 보고 **환급** ❌ | 2개월전 출역 확인 후 **공제** ✅ |

### Case 2: 기간 첫날 AND 마지막날 조건 추가

**시나리오**: 4개월전 기간 >= 8일, +30일 미출역, 기간 후 출역 없음, 기간 첫날&마지막날 출역

| | 기존 | 신규 |
|---|---|---|
| 판단 기준 | +30일 출역 여부만 | 첫날 AND 마지막날 출역 여부 |
| 결과 | **환급** | **공제** |

## 5. 구조 비교 요약

### 공제

| 항목 | 기존 | 신규 |
|---|---|---|
| 분기 수 | 7개 | 2개 |
| 크로스-월 윈도우 | ✅ 처리 | ❌ 없음 |
| 1/30-31 특수일 | ✅ 처리 | ❌ 없음 |
| 전전월 체크 | ✅ 있음 | ❌ 없음 |
| 코드 길이 | ~100줄 | ~25줄 |

### 환급

| 항목 | 기존 | 신규 |
|---|---|---|
| 리프 노드 수 | ~25개 | ~12개 |
| 서브 함수 수 | 8개 | 없음 |
| 1차 분기 기준 | 4개월전 유무 | 전체 출역 8일 기준 |
| 3개월전 기간 개념 | 없음 | 있음 (첫출역 +30일, from-start) |
| 확인 범위 | 3개월전 말일 | 2개월전 말일 |
| 중복 코드 | B, C 동일 패턴 반복 | 통합 처리 |
