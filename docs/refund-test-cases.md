## 케이스 12: sorted4.length >= 8 → 기간 후 출역 없어도 공제 정당 → 금액비교

> Step 5a 공제 정당 조건 ②`sorted4.length >= 8` 검증.  
> period4End 이후/당일 출역 없어도 sorted4 단독으로 8일 이상이면 공제 정당.

### 입력
```
[4개월전 - 2025년 12월] 8일
- 2025-12-20, 2025-12-21, 2025-12-22, 2025-12-23
- 2025-12-24, 2025-12-25, 2025-12-26, 2025-12-27

[3개월전 - 2026년 1월] 8일  ← 01-19(period4End) 포함
- 2026-01-05, 2026-01-06, 2026-01-07, 2026-01-08
- 2026-01-09, 2026-01-10, 2026-01-11, 2026-01-19

[2개월전 - 2026년 2월] 없음

공제 내역:
- 2026-01-08: ₩100,000
```

### 로직 흐름
1. 합산: sorted4(8) + sorted3(8) = **16일 >= 8** → 기간 계산
2. 4개월전 기간: 12월 20일 ~ 1월 19일 (12월=31일 → 12-20+30일=01-19)
   - 범위 내 출역: 12-20~27(8일) + 01-05~19(8일) = **16일 >= 8** → Step 5a
3. `sorted4.length(8) >= 8` ✓ → 조건 ② 충족, 단락평가로 이후 조건 미평가
4. 공제 정당. sorted5 없음, 공제 내역 있음 ✓
5. **결과: 금액비교 (AMT)**

> **참고**: 조건 ①(`allDates.some(d >= period4End)`) 단독 검증 케이스: sorted4를 7일 이하로 줄이고 sorted3에 01-19(period4End)를 포함하면 `d >= 01-19` 조건으로 공제 정당 진입.

## 케이스 13: Step 5c `>=` 엣지 — period3End 당일 출역 → 공제 정당 → 징수

> Step 5c는 `allDates.some(d >= period3End)` — period3End **당일 포함**.  
> Step 5a도 `some(d >= period4End)`로 통합됐으나, Step 5c는 추가로 특수일 조건이 없어 구조가 더 단순.

### 입력
```
[4개월전 - 2025년 12월] 없음  ← sorted4 = 0

[3개월전 - 2026년 1월] 8일
- 2026-01-05, 2026-01-06, 2026-01-07, 2026-01-08
- 2026-01-09, 2026-01-10, 2026-01-11, 2026-01-12

[2개월전 - 2026년 2월] 1일
- 2026-02-04  ← period3End 당일

공제 내역: 없음
```

### 로직 흐름
1. 합산: sorted4=0 → sorted3(8) + sorted2(1) = **9일 >= 8** → 기간 계산
2. period4Count = 0 < 8 → Step 5a 미진입
3. Step 5b: sorted4=0 → **통과** → Step 5c
4. Step 5c: period3Count
   - period3Start = 01-05, period3End = **02-04** (1월=31일 → 01-05+30일)
   - allDates = [01-05~12, 02-04], 범위 내([01-05, 02-04]) = **9일 >= 8**
5. `afterPeriod3`: allDates.some(d >= "02-04") → 02-04 >= 02-04 → **true** (당일 포함 `>=`)
6. 공제 정당. 공제 내역 없음 + sorted3(8) >= 8 → **징수**
7. **결과: 징수 (DEDUCT)**

> Step 5a·5c 모두 `some(d >= periodEnd)` 방식 — 당일 포함. sorted2에 02-05 이상이 없어도 02-04 당일 출역만으로 공제 정당.
